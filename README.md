# 1. 系统概述   

## 1.1 设计目的
本设计旨在开发一个功能完善的网络商城系统，专注于二次元周边商品的管理和销售。
系统需具备吸引二次元文化爱好者的界面设计，同时实现高效的商品管理、订单处理以及用户交互功能，以提升用户体验和促进商品流通。

## 1.2 项目概况
- 用户管理：支持用户注册、登录、个人信息修改及删除账户等功能。
- 商品管理：包括商品添加、编辑、删除及分类展示，支持上传商品图片、描述及库存管理。
- 购物车与订单处理：用户可将商品加入购物车，并完成结算生成订单，系统支持订单状态跟踪及历史订单查询。
- 支付集成：集成第三方支付接口，确保安全快速的在线支付体验。
- 评价系统：允许用户对购买的商品进行评价，增加互动性和商品可信度。

# 2. 系统分析

## 2.1 系统规划（需求规定）

- **前端界面**：响应式设计，适配不同设备。
- **后端技术**：使用Spring Boot作为后端框架，MySQL作为数据库。
- **安全性**：实现用户密码加密存储，使用HTTPS协议保障数据传输安全。

## 2.2 运行环境规定

- **服务器环境**：Linux操作系统，Apache Tomcat 9+作为应用服务器。
- **开发工具**：IntelliJ IDEA或Eclipse。
- **数据库环境**：MySQL 5.7+，需支持InnoDB存储引擎。

## 2.3 需求分析

- 用户需求：便捷的浏览、购买流程，个性化推荐。
- 商家需求：商品上架管理方便，实时销售数据分析。
- 系统需求：高并发处理能力，数据处理效率。

## 2.4 可行性分析

- **技术可行性**：基于成熟的Spring Boot框架及MySQL数据库，技术栈成熟稳定。
- **经济可行性**：开源软件减少初期投入，云服务降低运维成本。
- **操作可行性**：界面友好，管理后台简化日常操作。

# 3. 概要（总体）设计

## 3.1 系统功能设计

- 用户模块：注册/登录、个人信息管理、购物车管理、订单处理。
- 商品模块：商品展示、搜索、分类浏览、详情查看。
- 支付模块：支付接口集成、订单支付状态更新。
- 后台管理：商品上下架、用户管理、订单处理、数据分析。

## 3.2 系统结构设计

采用MVC架构，前端负责展示逻辑，后端处理业务逻辑，数据库存储数据。

## 3.3 数据库设计

- **users表**：存储用户信息，如用户名、密码、邮箱等。
- **products表**：存储商品信息，包括商品名、价格、库存、图片路径等。
- **orders表**：记录订单信息，如用户ID、商品ID、数量、总价、状态等。
- **reviews表**：用户评价，关联商品ID和用户ID。

# 4. 系统主要模块实现

## 4.1 登录验证

实现用户身份验证，包括账号密码的校验和权限控制。

## 4.2 功能

- 商品展示与筛选：通过Ajax异步加载商品列表，支持关键词搜索和分类筛选。
- 购物车：利用session或数据库保存购物车数据，实现商品添加、删除和数量调整。

## 4.3 编辑信息

提供用户资料编辑页面，允许用户修改个人信息，如地址、联系方式等。

# 5. 前台主要功能模块设计与实现

- **首页设计**：展示热门、新品商品，以及促销活动。
- **商品详情页**：展示商品图片、详情描述、用户评价和购买选项。
- **购物车模块**：实时更新购物车商品总价，支持一键清空和商品数量调整。
